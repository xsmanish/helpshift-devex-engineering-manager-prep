# Purpose

Engineering Manager interviews evaluate far more than technical capability.

Interviewers want evidence that you can:

* Lead teams
* Influence stakeholders
* Navigate ambiguity
* Handle conflict
* Drive execution
* Develop people
* Learn from failures

This document provides a framework for answering leadership and behavioral questions using real-world experiences.

---

# The STAR Framework

Always structure answers using:

```text
Situation

Task

Action

Result
```

---

# Better Version

For senior roles use:

```text
Situation

Challenge

Actions

Results

Lessons Learned
```

# Q5

# Tell Me About A Failure

---

## Important Rule

Choose a real failure.

Do not choose a disguised success.

---

### Example

Platform adoption slower than expected.

---

### Situation

New platform capabilities launched.

---

### Challenge

Teams continued using old processes.

---

### Actions

* Conducted user interviews
* Improved onboarding
* Enhanced documentation
* Added self-service capabilities

---

### Result

Adoption increased significantly.

---

### Lesson

Building technology is easier than driving change.

# Common Mistakes In Behavioral Interviews

| Mistake                     | Why It Fails           |
| --------------------------- | ---------------------- |
| Talking about team only     | Hides leadership       |
| Talking about yourself only | Ignores collaboration  |
| No metrics                  | Weak outcomes          |
| Blaming others              | Poor leadership signal |
| No lessons learned          | Low self-awareness     |
| Rambling                    | Poor communication     |

# Final Interview Formula

For every behavioral question:

```text
Situation

↓

Challenge

↓

Actions

↓

Results

↓

Lessons Learned
```

Always end with:

* Business Outcome
* Leadership Outcome
* Personal Learning

This consistently produces senior-level Engineering Manager answers rather than individual contributor answers.
